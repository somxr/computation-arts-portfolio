Dress Runway AR

I participated in a team for the Facebook September 2020 AR hackathon to create a fashion marketing tool. We built a world effect featuring an AR animated mannequin modelling a realistic fashion piece. I handled Meta Spark and Javascript development while my team did the 3D models on blender, Mixamo, and Marvelous Designer.

See Hackathon Submission Page

Concept

Concept

We wanted to give the user a better idea of how a garment looks in their real life 3D environment than just a photo or video online. We expect this could boost their confidence when making a buying decision.

Setup Interface

Setup Interface

First, the user is given a chance to make sure the ground is detected and the plane is resized appropriately. This is to avoid the model appearing suddenly in an unideal place while the user is still orienting the phone. I designed the interface and animated “runway floor” look using Spark’s visual shading nodes. The user can move the runway by swiping and panning then tap the screen to spawn the model.

Extra Features

Extra Features

The user can switch the mannequin’s poses and animations by tapping and holding the screen. My teammates provided me the animations using Mixamo and I set them up in the AR environment. I also implemented switching dress colors with Instagram’s native UI at the bottom of the screen.

Customize Lighting

Customize Lighting

It bothered us how the mannequin would look too bright in lower light situations. So, we gave the user the ability to adjust the virtual scene light’s brightness using Instagram’s slider UI. Automatic adjustment would be more ideal, but I managed to implement this simpler feature in the short time we had.

I learnt a lot during this project by working as part of a team. We brainstormed practical usecases of AR, whereas I was used to making purely artistic projects alone. We collectively found out the limits of social AR, which is restricted to small file sizes to accommodate user’s data usage. Another challenge is Spark’s ability to code using Javascipt as well as visual node graphs, and the interaction between the two. With each feature, we thought about which form of programming to use and how to combine them together. I hope to meet more like-minded people interested in working on big projects like this in the Computation Arts program.
